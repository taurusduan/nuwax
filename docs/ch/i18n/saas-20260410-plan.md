# Nuwax 前端多语言实施计划（最终版）

## Summary

1. 分支：`feat/i18n-frontend-saas-20260410`（基线：`feat-2026.4.10`）。
2. 机制：后端接口驱动（`GET /api/i18n/query`、`GET /api/i18n/lang/list`）。
3. 切换：语言保存成功后整页刷新，不引入前端运行时热切换状态机。
4. key 规范：`{Client}.{Scope}.{Domain}.{key}`。
5. 策略：`System.*` 一次性切换为新 key；存量页面采用“兜底翻译 + 核心页面显式改造”。

## Key Naming

1. `Client`：`NuwaxPC | NuwaxMobile | NuwaClaw`（PascalCase）。
2. `Scope`：`Pages | Components | Toast | Modal | Common`（PascalCase）。
3. `Domain`：业务域（PascalCase）。
4. `key`：语义化小驼峰（camelCase）。
5. 有效示例：`PC.Pages.Agent.pageTitle`。

## Implementation

1. L1（框架层）
2. i18n 运行时仅支持新 key；`System.*` 作为异常输入记录日志，不参与标准翻译流程。
3. 启动链路继续保留：`initI18n()` -> 首屏渲染；请求头透传 `Accept-Language` 与 `X-Lang`。
4. 失败兜底：缓存词典 -> 英文最小词典。
5. 新增治理能力：
6. 增量门禁：禁止新增硬编码中文、禁止新增 `System.*`、校验 `dict()` key 格式。
7. 全量盘点：按模块输出待改造清单 `docs/ch/i18n/saas-20260410-inventory.md`。

8. L2（组件/页面层）
9. 核心路径显式改造为 `dict(newKey)`：
10. 登录/验证码链路。
11. 主导航用户区与设置中心。
12. 系统配置-多语言管理页面。
13. Agent 编辑页头部关键交互文案。
14. 用户资料语言选择继续使用列表接口，保存通过 `saveUserLang(lang)`，成功后刷新。
15. 临时保存实现复用 `apiUserUpdate.lang`，后续专用接口上线只改 service 单点。

16. L3（后端渲染适配）
17. 后端返回文案直接展示。
18. 后端返回新规范 key（如 `PC.*`）由前端 `dict()` 解析。
19. 服务层统一适配字段变化，页面层不写兼容分支。

## 验收

1. 首屏按用户语言或后端默认语言渲染。
2. 语言切换成功后刷新生效；失败不刷新并提示。
3. `/api/i18n/query` 失败时缓存/英文兜底可用。
4. 新增代码满足 i18n 门禁规则。
