## 2. 变量引用向上查找逻辑增强 ✅

### 问题描述

- 条件节点在新建后没有配置任何值（初始状态）时会被错误命中
- 编辑开始节点后，其他节点无法正确引用其变量
- 变量引用标签显示 `undefined - 变量名` 格式

### 根本原因

**ID 类型不一致**：画布节点数据的 `id` 可能是字符串（如 `"10206"`），而代码中使用数字比较（`10206`），导致 Map 查找失败。

### 修复方案

#### `utils/variableReferenceV3.ts`

| 函数                   | 修改内容                                        |
| ---------------------- | ----------------------------------------------- |
| `buildNodeMap()`       | 将 `node.id` 转换为数字：`Number(node.id)`      |
| `previousNodes.push()` | 确保 `id` 是数字类型：`id: Number(predNode.id)` |
| `flattenArgsToMap()`   | 使用数字 ID 生成 key                            |
| 过滤逻辑               | 简化为检查 `outputArgs.length === 0`            |

#### `indexV3.tsx`

| 修改内容 |
| --- |
| `getReference()` 从画布 `graphRef.getNodes()` 直接获取节点数据，确保使用最新编辑状态 |
| 边数据也从画布 `graphRef.getEdges()` 获取 |

#### `models/workflow.ts`

| 函数        | 修改内容                        |
| ----------- | ------------------------------- |
| `getName()` | 使用 `Number(_id)` 转换后再比较 |

## 5. 变量聚合变量引用下拉 - 空节点项 Bug ✅

### 问题描述

变量聚合节点（`VariableAggregation`）配置面板中，变量引用下拉里某些上游节点菜单项可见，但展开后参数列表为空。

### 根本原因

`VariableSelector.tsx` 的 `getMenu()` 在 `nodes.map()` 中对每个上游节点调用 `filterOutputArgs()` 过滤可选参数：

- 已被同组其他 input 选走的参数 → 被 `filter()` **直接移除**（`selectedKeys` 集合命中）
- 类型不匹配的参数 → 标记 `disabled` 保留显示

当分组内的多个 input 已将同一个上游节点的所有参数都选走后，该节点的 `filteredOutputArgs` 变为空数组。但原代码未对此判空，仍把该节点渲染成菜单项，导致：

- 下拉中节点名称（菜单项）出现
- 展开后 `<Tree>` 无内容

### 修复方案

**文件**：`component/VariableAggregation/VariableSelector.tsx`

最小 diff（2 行）：

```diff
  return nodes.map((node) => {
    const filteredOutputArgs = filterOutputArgs(...);

+   if (!filteredOutputArgs.length) return null;

    return { key: node.id, ... };
- });
+ }).filter(Boolean);
```

不涉及其他文件，原始 map 结构保持不变，侵入最小。
